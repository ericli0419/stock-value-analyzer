# 估值定量手册（Valuation Models）

> 配套 `stock-value-analyzer` v2.0 使用。本手册把邱国鹭"好价格/安全边际"从**拍脑袋 PE** 升级为**可复现的内在价值模型**。
> 核心模型（反向 DCF 隐含增速）由 `scripts/value_models.py` 自动计算并写入 `models.reverse_dcf`；
> 情景估值/EPV/分行业路由由 AI 按本手册口径在报告中显式列出计算过程。
>
> **核心理念**：估值不是预测股价，而是**回答两个问题**——
> **① 当前价格隐含了什么样的未来（反向 DCF）？这个未来现实吗？**
> **② 在保守/中性/乐观三种未来下，我亏的概率和幅度有多大（情景概率加权）？**

---

## 一、反向 DCF（Reverse DCF）— 本手册的锚（强制）

### 1.1 思想

不去预测增速（正向 DCF 易沦为"为结论凑假设"），而是**反解**：
> 当前市值，隐含了市场对未来自由现金流多高的复合增速预期？

把这个**隐含增速**与公司**历史 CAGR、分析师一致预期、行业天花板**对比，判断贵贱：

| 隐含增速 vs 历史/可实现增速 | 结论 |
|---|---|
| 隐含增速 **显著低于**历史可持续增速 | 🟢 可能低估（市场预期过悲观） |
| 隐含增速 **约等于**历史增速 | 🟡 定价合理 |
| 隐含增速 **显著高于**历史增速且无新驱动 | 🔴 高估（需极强增长才撑得起） |

### 1.2 脚本口径（`models.reverse_dcf`）

两阶段模型，二分法反解高速增长期增速 `g`：

```
市值 = Σ_{t=1..N} FCF₀·(1+g)ᵗ / (1+r)ᵗ  +  [FCF_N·(1+g_term)/(r-g_term)] / (1+r)^N
默认：N=10年，r(折现率)=9%，g_term(永续增速)=3%，FCF₀=最新自由现金流
```

- 输出 `implied_growth`（隐含增速）。`base_fcf` 来自 yfinance `freeCashflow` 或 `CFO+capex`。
- **边界提示**：若 `implied_growth=None` 且 note 含"极低/极高"，说明现价处于极端区间。
- **不适用**：亏损/负 FCF（脚本返回 `applicable=False`），改用 EPV 或 PS/PB 路由。

### 1.3 报告呈现

> 必须写出三组对比数：**隐含增速 / 近 5 年营收(或净利)CAGR / 分析师未来 3 年预期增速**，并下结论贵贱。

## 二、情景概率加权估值（强制）

单点目标价是伪精确。用**牛/中/熊三情景 × 概率**得到**期望价值**与**下行价值**。

### 2.1 模板

| 情景 | 关键假设（增速/利润率/估值倍数） | 对应每股价值 | 概率 |
|---|---|---|---|
| 牛市 | 乐观但可论证 | V_bull | p_bull |
| 中性 | 基准（贴近历史/一致预期） | V_base | p_base |
| 熊市 | 保守（增速放缓+杀估值） | V_bear | p_bear |

```
期望价值 E[V] = V_bull·p_bull + V_base·p_base + V_bear·p_bear   （Σp = 1）
下行价值      = V_bear（用于安全边际下限）
安全边际      = (E[V] - 现价) / E[V]
下行保护      = (V_bear - 现价) / 现价   （负值=熊市仍要亏多少）
```

### 2.2 决策含义

- **期望价值 > 现价 且 熊市跌幅可承受** → 风险收益比佳。
- **期望价值 > 现价 但 熊市 -50%** → 期望虽正，但下行过大，需更大安全边际或更小仓位。
- 概率分配需在报告中说明依据（避免为结论凑概率）。

---

## 三、EPV 盈利能力价值（Earnings Power Value，零增长锚）

格雷厄姆-格林沃尔德思想：**先不给增长付钱**，看现有盈利能力值多少。

```
调整后 EBIT → NOPAT = 正常化EBIT × (1 - 税率)
EPV(企业价值) = NOPAT / WACC
EPV(股权)     = EPV企业价值 - 净负债
```

- **正常化 EBIT**：用 5-10 年平均 EBIT 利润率 × 当前营收（熨平周期），见 `time-series-analysis.md`。
- **用途**：`EPV股权 vs 当前市值` —— 若市值 < EPV，则市场**倒贴**增长，安全边际厚；若市值 >> EPV，则股价**全部靠增长预期**支撑，脆弱。
- **资产价值对照**：重资产/周期股可同时算重置成本/清算价值作为地板。

## 四、分行业估值路由（强制：选错模型 = 误判）

> **铁律**：不能跨行业比 PE，更不能用统一模型套所有行业。先路由，再估值。

| 行业类型 | 首选模型 | 禁用/慎用 | 关键口径 |
|---|---|---|---|
| **金融（银行/保险/券商）** | **PB-ROE**、股息折现(DDM) | ❌ 标准 DCF、❌ 毛利率、❌ EV/EBITDA | 合理 PB ≈ (ROE-g)/(COE-g)；看不良率/拨备覆盖 |
| **周期（资源/化工/航运/地产）** | **归一化盈利 PE**、市值/产能或储量、PB | ❌ 用景气顶点 PE | 必须用穿越周期的平均利润率，见时序手册 |
| **成长（科技/医药/平台）** | **反向 DCF**、PEG、Rule of 40 | ❌ 静态低 PE 当便宜 | PEG<1 优；Rule of 40：营收增速+利润率≥40% |
| **稳定消费/公用事业** | **DCF/DDM**、FCF 收益率 | — | FCF 收益率 vs 国债利率比价 |
| **重资产制造** | **EV/EBITDA**、EPV、PB | 慎用 PS | 关注 ROIC vs WACC（见财务质量手册） |
| **亏损但有收入** | **PS**、用户/单位经济模型 | ❌ PE（无意义） | PS 须配毛利率与盈利路径 |

### 4.1 金融股 PB-ROE 速算

```
合理 PB ≈ (可持续ROE - 永续增速g) / (权益成本COE - g)
当前 PB < 合理 PB → 低估；> 合理 PB → 高估
```

### 4.2 周期股归一化

用 5-10 年平均利润率重算"正常化盈利"，再给周期中枢 PE。**景气高点低 PE 往往是顶部陷阱**（见 `valuation-traps.md`）。

## 五、安全边际计算口径（统一）

```
安全边际 = (内在价值估计 - 当前价格) / 内在价值估计
内在价值估计 = 多模型交叉（反向DCF反推 + 情景期望价值 E[V] + EPV）取审慎值
```

| 安全边际 | 评分指向（估值模块 0-25 分映射见 SKILL.md） |
|---|---|
| > 40% | 极厚（20-25） |
| 20-40% | 较好（15-20） |
| 0-20% | 偏薄（5-15） |
| < 0% | 高估（0-5） |

> **审慎原则**：多模型结论分歧大时，取**偏保守**者；并以情景熊市价 `V_bear` 作为下行参照，而非只看期望值。

---

## 六、估值模块工作流（报告必走）

```
1. 行业路由 → 确定首选估值模型（第四节表）
2. 反向 DCF → 读取 models.reverse_dcf.implied_growth，与历史CAGR/一致预期三对比
3. 情景概率加权 → 列牛/中/熊表，算 E[V]、V_bear、下行保护
4. EPV（适用时）→ 判断"现价里有多少是为增长付的钱"
5. 历史估值分位 → 当前 PE/PB 在 5 年区间的位置（结合时序手册）
6. 综合 → 安全边际 + 估值陷阱检查（valuation-traps.md）→ 估值模块得分
```

```
models.reverse_dcf.{applicable, implied_growth, assumptions{discount_rate, terminal_growth, high_growth_years, base_fcf, market_cap}}
models.growth_consistency.{revenue_cagr, net_income_cagr}   # 反向DCF的对比基准
```

> 反向 DCF 隐含增速是脚本算的"硬数"；情景/EPV/路由由 AI 按本手册口径计算，**所有算式必须在报告中可见并经 Step 8 算术回验**。
