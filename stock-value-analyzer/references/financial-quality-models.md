# 财务质量与造假识别量化手册（Financial Quality & Forensic Models）

> 配套 `stock-value-analyzer` v2.0 使用。本手册定义**可复现**的财务质量评分与造假/破产风险模型。
> 所有指标由 `scripts/value_models.py` 自动计算并写入取数 JSON 的 `models` 字段；报告**直接引用脚本输出**，不得手算口径冲突。
>
> **核心理念**：邱国鹭"好公司"的定性判断（护城河、管理层）需要量化骨架支撑。
> ROE 高不等于真好（可能靠杠杆/造假堆出来）；本手册回答三问——
> **① 这个 ROE 是怎么来的（杜邦）？② 它真创造价值吗（ROIC vs WACC）？③ 它是不是装出来的（F/M/Z + 盈余质量）？**

---

## 〇、量化层与定性层的分工

| 层 | 工具 | 回答的问题 | 性质 |
|---|---|---|---|
| **量化层** | 本手册 6 个模型 | "数字上真好/假好？有没有造假破产风险？" | 客观锚，可复现 |
| **定性层** | 三好原则 + 护城河指南 | "护城河能不能持续？管理层可不可信？" | 人文判断 |

**铁律**：量化层是**否决与降级**工具（发现造假/破产/价值毁灭即扣分或一票否决），不是**加分**工具。
数字漂亮只是入场券，能不能买仍由定性层 + 估值层决定。

---

## 一、杜邦分解（DuPont Analysis）— ROE 的来源拆解

### 1.1 三因子（基础）

```
ROE = 净利率 × 资产周转率 × 权益乘数
     = (净利润/营收) × (营收/总资产) × (总资产/股东权益)
```

### 1.2 五因子（区分经营与杠杆）

```
ROE = 税负担 × 利息负担 × EBIT利润率 × 资产周转率 × 权益乘数
税负担   = 净利润 / 税前利润      （税收侵蚀，越接近1越好）
利息负担 = 税前利润 / EBIT        （利息侵蚀，越接近1越好）
EBIT利润率 = EBIT / 营收          （经营盈利能力）
资产周转率 = 营收 / 总资产        （资产效率）
权益乘数  = 总资产 / 股东权益      （财务杠杆）
```

### 1.3 判读规则（脚本字段 `models.dupont.components`）

| 信号 | 解读 | 评分指向 |
|---|---|---|
| ROE 高 + 权益乘数 ≤ 2 + 净利率高 | **优质经营驱动**（茅台型） | 财务质量加分 |
| ROE 高 + 权益乘数 > 3 | **杠杆驱动**，需查行业（银行/地产正常，制造业危险） | 中性/警惕 |
| ROE 高 + 资产周转率高 + 净利率低 | **薄利多销型**（零售/家电），看周转持续性 | 中性 |
| 利息负担 < 0.7 | 利息严重侵蚀利润，债务负担重 | 财务健康扣分 |

> 脚本对权益乘数 > 3 自动写入 `note`：「ROE 含较多杠杆驱动成分」。

## 二、ROIC vs WACC — 是否真正创造价值

```
NOPAT = EBIT × (1 - 有效税率)
投入资本 = 有息负债 + 股东权益 - 现金及等价物
ROIC = NOPAT / 投入资本
```

- **价值创造判据**：`ROIC > WACC` → 创造价值；`ROIC < WACC` → 毁灭价值（增长越快毁得越多）。
- 脚本默认参考 `WACC = 9%`（写入 `models.roic.reference_wacc`），并给出 `value_creation` 布尔。
- **WACC 精修（人工，报告中可选）**：`WACC = E/V × Re + D/V × Rd × (1-t)`，其中 `Re = Rf + β×(Rm-Rf)`。
  成熟蓝筹常落 8-10%，高 β 成长股 10-13%。脚本默认 9% 仅作快速锚，报告可按个股调整后复核结论。

| ROIC 水平 | 解读 |
|---|---|
| 持续 > 15% | 强护城河的量化印证（高资本回报难以被竞争抹平） |
| 10-15% | 良好 |
| 接近或低于 WACC | 缺乏定价权/重资产拖累，增长不创造价值 |

> **与定性护城河交叉验证**：若定性判定"宽护城河"但 ROIC 长期 < WACC → 护城河结论存疑，必须复核。

## 三、Piotroski F-Score（9 分制基本面健康度）

衡量基本面**边际改善**方向，需当期与上期两年数据。脚本输出 `models.piotroski_f`。

| 维度 | # | 得分项（满足记 1 分） |
|---|---|---|
| **盈利能力(4)** | 1 | ROA > 0 |
| | 2 | 经营现金流 CFO > 0 |
| | 3 | ROA 同比上升 |
| | 4 | CFO > 净利润（应计质量好，利润有现金支撑） |
| **杠杆/流动性(3)** | 5 | 长期负债率同比下降 |
| | 6 | 流动比率同比上升 |
| | 7 | 未增发稀释股本 |
| **经营效率(2)** | 8 | 毛利率同比上升 |
| | 9 | 资产周转率同比上升 |

| F-Score | 评级 | 含义 |
|---|---|---|
| 7-9 | 强 | 基本面健康且在改善 |
| 4-6 | 中 | 中性 |
| 0-3 | 弱 | 基本面恶化，警惕 |

> ⚠️ 脚本对缺失数据项默认计 0 分。若 `breakdown` 中多项为 0 且对应数据缺失，分数会被低估，**报告须结合 breakdown 逐项核实**，不可只看总分。

## 四、Beneish M-Score（盈余操纵识别）

8 变量模型，识别**财务报表操纵**可能性。脚本输出 `models.beneish_m`。

```
M = -4.84 + 0.92·DSRI + 0.528·GMI + 0.404·AQI + 0.892·SGI
        + 0.115·DEPI - 0.172·SGAI + 4.679·TATA - 0.327·LVGI
```

| 变量 | 含义 | 红旗方向 |
|---|---|---|
| DSRI | 应收账款/营收 同比变化 | ↑ 可能虚增收入 |
| GMI | 毛利率恶化指数 | ↑ 恶化诱发操纵动机 |
| AQI | 资产质量指数（非流动非PPE资产占比变化） | ↑ 费用资本化嫌疑 |
| SGI | 营收增长指数 | ↑ 高增长伴随操纵动机 |
| DEPI | 折旧率下降指数 | ↑ 放缓折旧虚增利润 |
| SGAI | 销管费/营收 变化 | — |
| TATA | 总应计/总资产 = (净利-CFO)/总资产 | ↑ 利润缺现金支撑（权重最高） |
| LVGI | 杠杆变化指数 | — |

- **阈值**：`M > -1.78` → **可能存在盈余操纵（红旗）**；`M < -1.78` → 未触发。
- **适用范围**：**仅非金融制造/商业企业**。金融股脚本自动返回 `applicable=False`。
- **降级规则**：缺失变量脚本用中性值（对比类=1、TATA=0）代入并记入 `defaults_used`；
  **若缺失 ≥ 5 个变量，脚本返回 `applicable=False`（结果不可靠，禁用）**。

## 五、Altman Z-Score（破产风险预警）

```
Z = 1.2·X1 + 1.4·X2 + 3.3·X3 + 0.6·X4 + 1.0·X5
X1 = 营运资本/总资产    X2 = 留存收益/总资产    X3 = EBIT/总资产
X4 = 股权市值/总负债    X5 = 营收/总资产
```

| Z 值 | 区域 | 含义 |
|---|---|---|
| > 2.99 | 安全 | 破产风险低 |
| 1.81-2.99 | 灰色 | 需警惕 |
| < 1.81 | 财务困境 | 高破产风险 |

- **适用范围**：非金融制造与商业企业；金融股/缺关键科目时脚本返回 `applicable=False`。
- 脚本输出 `models.altman_z`，含五个 X 分量便于复核。

## 六、盈余质量（Earnings Quality）

```
现金含量(cash_conversion) = 经营现金流 / 净利润
应计比(accrual_ratio)     = (净利润 - 经营现金流) / 总资产
```

| 现金含量 | 评级 | 含义 |
|---|---|---|
| ≥ 1.0 | 优 | 利润全部有现金支撑 |
| 0.8-1.0 | 良 | — |
| 0.5-0.8 | 一般 | 部分利润是应计，关注 |
| < 0.5 | 差 | 利润含金量低，警惕"纸面富贵" |

- 应计比越高 → 盈余质量越差（与 M-Score 的 TATA 同源逻辑）。
- 脚本输出 `models.earnings_quality`。

## 七、🚫 接入一票否决 / 强制降级的规则（v2.0 核心）

> 量化造假/破产模型**触线即进入 SKILL.md 一票否决或强制降级**，但**必须先确认模型 `applicable=True`**，避免误杀金融股/轻资产标的。

| 触发条件 | 处置 | 前置确认 |
|---|---|---|
| Beneish `M-Score > -1.78` 且 `applicable=True` 且 `defaults_used` 为空/极少 | **一票否决候选**：标记"盈余操纵高风险"，需人工追溯财报原文，未排除则结论降为"不推荐" | 必须非金融、变量完整 |
| Altman `Z-Score < 1.81` 且 `applicable=True` | **强制降级**：财务健康分清零方向，结论降一档；连续2年 < 1.81 → 一票否决候选 | 必须非金融制造/商业 |
| 现金含量连续 3 年 < 0.5 | 盈余质量"差"，公司分财务模块大幅扣分 | — |
| ROIC 连续 3 年 < WACC | 价值毁灭，护城河结论存疑，公司分降级 | 已按个股复核 WACC |
| F-Score ≤ 2 且数据完整 | 基本面恶化，结论谨慎 | breakdown 非缺失导致 |

**反误杀原则**：任何 `applicable=False` 的模型**不得作为否决依据**，仅在报告中注明"该模型不适用，已转定性判断"。
